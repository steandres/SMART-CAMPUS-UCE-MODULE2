# Frontend Demo - Student Welfare and Support Module

This guide explains how to demonstrate the `welfare-frontend` integration with the existing backend microservices.

## 1. Start The Full Stack

From the monorepo root, run:

```bash
docker compose up -d --build
```

Wait until all containers are running.

Expected services:

- `welfare-frontend`: `http://localhost:3002`
- `scholarship-service`: `http://localhost:3000`
- `socioeconomic-form-service`: `http://localhost:3001`
- `postgres`: `localhost:5432`
- `mongo`: `localhost:27017`

## 2. Open The Dashboard

Open:

```text
http://localhost:3002
```

Verify that the dashboard displays:

- `Student Welfare and Support Module`
- `Scholarship Requests`
- `Socioeconomic Forms`
- `Pending Reviews`
- `Approved Scholarships`

## 3. Verify Health Services

On the dashboard, verify the service status panel.

Expected services:

- `scholarship-service health`
- `socioeconomic-form-service health`

You can also verify manually:

```text
http://localhost:3000/health
http://localhost:3001/health
```

Expected response:

```json
{
  "status": "ok"
}
```

## 4. Create A Scholarship

Open:

```text
http://localhost:3002/scholarships
```

Fill the form with test data:

```text
studentId: 2f6f25c5-5df3-45e7-8f6f-3396a3ac4cf5
scholarshipType: ECONOMIC_SUPPORT
reason: Financial hardship due to family situation
status: PENDING
```

Click:

```text
Create Scholarship
```

Expected result:

- Success message is displayed.
- The scholarship appears in the table.
- The `id` is generated by the backend.

## 5. Approve A Scholarship

In the scholarship table, click:

```text
Approve
```

Expected result:

- Success message is displayed.
- The scholarship status changes to `APPROVED`.

You can also demonstrate rejection by creating another request and clicking:

```text
Reject
```

Expected status:

```text
REJECTED
```

## 6. Create A Socioeconomic Form

Open:

```text
http://localhost:3002/socioeconomic-forms
```

Fill the form with test data:

```text
studentId: 4d51ffba-3cc1-410d-82d3-92ef2c8a6104
familyIncome: 550
housingType: RENTED
familyMembers: 4
employmentStatus: UNEMPLOYED
vulnerabilityFactors: Single-parent household
observations: Student requires socioeconomic support evaluation
```

Click:

```text
Create Form
```

Expected result:

- Success message is displayed.
- The form appears in the table.
- The `id` is generated by the backend.

## 7. Search By Student ID

In the socioeconomic forms screen, use the search input:

```text
4d51ffba-3cc1-410d-82d3-92ef2c8a6104
```

Click:

```text
Search
```

Expected result:

- The table shows the matching socioeconomic form.

## 8. Evidence Frontend-Backend Integration

To demonstrate integration, explain the following:

- The frontend runs as a Next.js application.
- It consumes `scholarship-service` using `NEXT_PUBLIC_SCHOLARSHIP_API_URL`.
- It consumes `socioeconomic-form-service` using `NEXT_PUBLIC_SOCIOECONOMIC_API_URL`.
- Scholarship data is persisted through PostgreSQL.
- Socioeconomic form data is persisted through MongoDB.
- The dashboard health panel calls both `/health` endpoints.
- The management screens call REST endpoints for create, read, update, and delete operations.
